第 1 章：CSS基础语法

CSS要解决的问题

在CSS发明之前HTML标签既要用于组织内容还要处理布局、样式等问题，结果导致HTML结构混乱，无法维护。CSS产生的原因就是要把HTML从它不擅长的布局、样式中解放出来，从而使HTML专注于内容。而CSS在样式方面的强大功能极大的提升了网页的显示效果，提高了开发效率。

因此有了CSS之后，我们写HTML代码时应只关心如何组织内容，比如使用p标签来表示一个段落，h1用来表示一级标题，而不关心HTML如何显示，这就意味着写HTML的人不应该在HTML结构中加上id, class等属性。然后我们为这个HTML页面应用CSS，来实现这个页面的视觉效果。

CSS放在哪里

CSS按其位置可以分成三种：

  • 内嵌样式(Inline Style)

    内嵌样式就是把CSS写在HTML的style属性里。<h1 style="color: red">H1 Title</h1>

  • 内部样式表(Internal Style Sheet)

    内部样式表主要是将CSS代码放到head里的style标签里，例如：

    <head>
  <style type="text/css">
    h1 {
      color: red;
      font-size: 30px;
    }
  </style>
</head>

  • 外部样式表(External Style Sheet)

    外部样式表是指将CSS放到一个CSS文件里，通常是以.css结尾的文件，然后在HTML文件里将该CSS文件引入进来。<link rel="stylesheet" href="bootstrap.css">

通常我们推荐外部样式表，这样做目的有以下优点：

  1. 可以CSS代码和HTML代码彻底分离开来，便于代码的开发、维护。
  2. 多个HTML文件可以共享一个CSS文件，有助于代码复用。
  3. 浏览器可以缓存CSS文件，有利于前端性能优化。

内联样式多用于JavaScript通过DOM修改样式，而内部样式多用于对显示速度要求极高的页面，因为内部样式要比外部样式文件下载速度快很多，这样可以保证在CSS载入时即时渲染，通常各门户网站会这么做，例如网易。

CSS 基础语法

CSS 规则由两个主要的部分构成：选择器，以及一条或多条样式声明。

selector {property: value; property: value;...}

selector(选择器)用于查找要应用CSS样式的元素，而后面的花括号里则是具体的CSS样式列表，每一个以property: value;的形式出现，属性（property）是您希望设置的样式属性（style attribute）。每个属性有一个值，属性和值被冒号分开。每一对属性和值以分号隔开。例如：我们需要给所有的h2元素设置字体大小为28px，颜色为红色：

h2 {
  color: red;
  font-size: 28px;
}

选择器介绍

上面说过，选择器的目的是找到特定的HTML元素，选择器可以用HTML标签，ID，Class以及它们的组合，下面是一些例子：

  • p {font-family: "sans serif"}，HTML元素标签选择器，选中所有的P元素
  • h1,h2,h3,h4,h5,h6 {color: green}，分组选择器，h1到h6等6个元素标签拥有相同的样式
  • .right {text-align: right}，类选择器，选中所有class值为right的元素
  • #green {color: green}，ID选择器，选中id值为green的元素
  • p.right div {}，组合选择器，选中拥有class值为right的所有P标签下面所有div元素
  • #header h2 {}，组合选择器，选中拥有id值为header元素下面所有的h2元素标签

多个选择器用逗号隔开表示一组元素拥有相同的CSS样式，可以避免重复代码。选择器中间有空格表示前后两个选择器是祖先和子孙的关系，例如#header h2表示要选择#header下面所有h2元素。如果两个选择器中间没有空格则表示它们同一层次，没有上下级关系，比如p.right，表示要查找拥有class值为right的P元素。

详细的CSS选择器用法请参考http://blog.csdn.net/borishuai/article/details/8636377

ID和Class

ID和Class两个属性经常用于CSS选择器中，好多初学者分不清楚什么时候用ID，什么时候用Class。下面是它们的特性：

  1. ID，顾名思义就是identity，也就是身份证的意思，所以ID在一个页面里是不能有重复的。其实，对于CSS来讲，即使有重复的ID也能正常工作的，即选择器能够找到所有重复的ID，但是JavaScript却无法正常工作，例如：document.getElementById()方法找到第一个ID就不会往下找了。
  2. Class，顾名思义就是classification，也就是分类的意思，即一个Class即代表某种分类，例如：.error，所有的错误消息都加上这个Class。这就说明Class是可以重复的。
  3. 一个元素至多拥有一个ID，就像人只能拥有一个身份证号码一样。
  4. 一个元素可能拥有多个不同的Class，因为一个Class代表一个分类或类别，那么一个元素就可能属于多个分类。就像人可能同时喜欢篮球、足球、乒乓球。例如：<div class="basketball football ping-pong"></div>，多个Class用空格隔开。

通过以上的特征来看，当需要为某个特定的HTML元素应用样式时，可以选择使用ID，而当为一组拥有相同特征的元素应有样式时，则应该使用Class。

分组选择器(h1, h2, h3)和Class选择器行为上有点相似，即一次可以选中多个元素，但是语义上是完全不同的。分组选择器表示多个元素在某个时间点上拥有相同的样式，过会可能就不相同了，但是Class表示多个元素拥有相同的分类或者相同的属性。样式是为分类或属性服务的。
